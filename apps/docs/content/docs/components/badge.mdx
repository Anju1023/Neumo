# Badge

ニューモフィズム2.0デザインのバッジコンポーネントです。ステータス、カテゴリ、カウントなどを表示するための小さなラベルとして使用します。

<Callout type="info" title="用途">
Badgeは状態やカテゴリを視覚的に伝えるために使用します。色分けされた5種類のバリアントで、さまざまな意味を表現できます。
</Callout>

## インポート

```tsx
import { Badge } from 'neumo-ui';
```

## 基本的な使い方

<ComponentPreview code={`<Badge>Default</Badge>`}>
  <Badge>Default</Badge>
</ComponentPreview>

## アクセシビリティ

<Callout type="info" title="色だけに頼らない">
色覚特性を持つユーザーのために、Badgeの意味を色だけでなくテキストでも伝えることが重要です。例えば、「Success」バリアントには「完了」「OK」などの明確なテキストを入れてください。
</Callout>

<Callout type="warn" title="動的なステータス">
Badgeの内容が動的に変化する場合（例：通知カウント）、`aria-live`を使用してスクリーンリーダーに変更を通知することを検討してください。
</Callout>

## ベストプラクティス

<Accordions>
  <Accordion title="適切なバリアントの選択">
    - **success**: 肯定的な結果、完了状態、アクティブ状態
    - **warning**: 注意が必要な状態、保留中、期限間近
    - **error**: 否定的な結果、エラー、ブロック状態
    - **info**: 新着情報、ベータ機能、お知らせ
    - **default**: 上記に当てはまらない一般的なラベル
  </Accordion>
  <Accordion title="テキストは簡潔に">
    Badgeのテキストは1〜2語程度に抑えてください。長いテキストには適していません。
  </Accordion>
  <Accordion title="配置のヒント">
    - カードやリストアイテムの右上に配置すると視認性が高い
    - アバターやボタンと組み合わせる場合は、`position: absolute`で重ねて配置
    - 複数のBadgeを並べる場合は、一貫した順序を保つ
  </Accordion>
</Accordions>
